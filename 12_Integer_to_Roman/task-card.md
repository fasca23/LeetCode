**Подход: Жадное вычитание с таблицей номиналов**

1. **Идея:** Имеем таблицу соответствий чисел и римских символов, отсортированную по убыванию номинала, включая пары для правил вычитания (900, 400, 90, 40, 9, 4). Для заданного числа жадно вычитаем максимально возможный номинал, добавляя его римское представление к результату, пока число не станет нулём.
2. **Логика:** Создаём список кортежей `vals = [(1000, 'M'), (900, 'CM'), (500, 'D'), (400, 'CD'), (100, 'C'), (90, 'XC'), (50, 'L'), (40, 'XL'), (10, 'X'), (9, 'IX'), (5, 'V'), (4, 'IV'), (1, 'I')]`. Результат — пустая строка `res = ""`. Для каждой пары `(val, sym)` в `vals`: пока `num >= val`: `res += sym`, `num -= val`. Возвращаем `res`.
3. **Проверка переполнения:** Не требуется, входное число ограничено диапазоном от 1 до 3999 по условию задачи.
4. **Почему жадный подход работает:** Таблица включает все "вычитательные" пары (CM = 900, CD = 400, XC = 90, XL = 40, IX = 9, IV = 4). Римская система не требует размена — всегда можно взять максимальный номинал без риска получить неверное представление. Без этих пар жадный подход не сработал бы (например, 4 дало бы IIII вместо IV).
5. **Особые случаи:** `num = 3` → `"III"` (I + I + I). `num = 58` → `"LVIII"` (50 + 5 + 1 + 1 + 1). `num = 1994` → `"MCMXCIV"` (1000 + 900 + 90 + 4). `num = 4` → `"IV"` (не IIII). `num = 9` → `"IX"` (не VIIII). `num = 40` → `"XL"` (не XXXX). `num = 3999` → `"MMMCMXCIX"`.
6. **Время:** O(1) — количество итераций ограничено константой: 13 номиналов, каждый вычитается не более ~3 раз (например, 3 раза по 1000 для 3000). Всего не более 15–20 операций.
7. **Память:** O(1) — таблица фиксированного размера (13 пар) + строка результата длиной не более 15 символов (для 3999).

**Ключевой момент:** Включение в таблицу номиналов 900, 400, 90, 40, 9, 4 — это и есть учёт римских правил вычитания. Именно они делают жадный алгоритм корректным для всех чисел.
