**Подход: Бинарный поиск по меньшему массиву**

1. **Идея:** Делим оба массива так, чтобы левая половина содержала ровно `total//2` элементов, и все элементы слева были ≤ всех элементов справа. Медиана — либо `min(правой)`, либо среднее `max(левой)` и `min(правой)`.
2. **Логика:** Бинарный поиск по `mid1` (сколько из nums1 берём слева). `mid2 = half - mid1` (остальные из nums2). Граничные элементы: `l1, r1, l2, r2`. Если `l1 ≤ r2` и `l2 ≤ r1` — разбиение верное.
3. **Почему меньший массив:** Бинарный поиск по меньшему даёт O(log(min(n, m))). По большому было бы O(log(max(n, m))).
4. **Особые случаи:** Пустой массив → `float('-inf')` / `float('inf')` для границ. Нечётный total → `min(r1, r2)`. Чётный → `(max(l1,l2) + min(r1,r2)) / 2`.
5. **Время:** O(log(min(n, m))) — бинарный поиск.
6. **Память:** O(1) — только переменные.

**Ключевой момент:** Границы `l1, r1, l2, r2` — это последний элемент левой части и первый элемент правой части каждого массива. Условие корректности: `l1 ≤ r2` и `l2 ≤ r1`.
