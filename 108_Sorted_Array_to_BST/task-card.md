**Подход: Рекурсивное построение с выбором середины**

1. **Идея:** Чтобы BST был сбалансированным, корень должен быть серединой массива. Тогда левое и правое поддеревья будут примерно равного размера. Рекурсивно применяем тот же принцип к половинам.
2. **Логика:** `mid = (left + right) // 2`. `root = TreeNode(nums[mid])`. `root.left = _build(left, mid-1)`. `root.right = _build(mid+1, right)`. База: `left > right` → None.
3. **Почему именно середина:** Если брать первый или последний элемент — получится вырожденное дерево (цепочка). Середина гарантирует сбалансированность.
4. **Время:** O(n) — каждый элемент один раз становится узлом.
5. **Память:** O(log n) — стек рекурсии для сбалансированного дерева. Для вырожденного было бы O(n), но оно и не получается.

**Ключевой момент:** `mid = (left + right) // 2` — выбор середины. Именно это делает дерево сбалансированным.
