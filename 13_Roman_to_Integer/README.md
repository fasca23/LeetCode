**Подход: Обход справа налево**

1. **Идея:** Идём по строке справа налево, сравнивая текущий символ с предыдущим (более правым). Если текущий меньше — вычитаем, иначе складываем.
2. **Логика:** Для каждого символа берём его значение из словаря. Если `cur < prev` (например, I перед V в "IV") — вычитаем: `total -= cur`. Иначе (VI, III) — складываем: `total += cur`. Обновляем `prev = cur`.
3. **Почему справа налево:** При обходе слева направо нужно заглядывать вперёд чтобы понять, вычитать или складывать. Справа налево мы всегда знаем предыдущее значение и решение принимается мгновенно.
4. **Особые случаи:** Вычитание работает только для пар: IV=4, IX=9, XL=40, XC=90, CD=400, CM=900. В остальных случаях символы просто складываются.
5. **Время:** O(n) — один проход по строке.
6. **Память:** O(1) — словарь фиксированного размера (7 символов).

**Ключевой момент:** `for ch in reversed(s)` — обход справа налево позволяет сравнивать текущий символ с уже обработанным правым и сразу определять операцию.
