**Подход: Пошаговая обработка строки с проверкой границ**

1. **Идея:** Идём по строке слева направо, игнорируем начальные пробелы, считываем знак, затем собираем цифры в число до первого нецифрового символа. При выходе за 32-битные границы возвращаем граничное значение (clamping).
2. **Логика:** Индекс `i = 0`. Пропускаем пробелы: `while i < n and s[i] == ' ': i += 1`. Если `i == n` — цифр нет, возвращаем 0. Определяем знак: `sign = 1`. Если `s[i] == '-'`: `sign = -1, i += 1`. Иначе если `s[i] == '+'`: `i += 1`. Результат `res = 0`. Пока `i < n` и `s[i]` — цифра: `digit = ord(s[i]) - ord('0')`. Проверяем переполнение. Если нет — `res = res * 10 + digit`, `i += 1`. В конце `return sign * res`.
3. **Проверка переполнения:** ДО умножения `res * 10` проверяем: `if res > (INT_MAX - digit) // 10`. Если переполнение — возвращаем `INT_MAX` (2147483647) при `sign == 1` или `INT_MIN` (-2147483648) при `sign == -1`. Это называется clamping, в отличие от задачи 7, где возвращается 0.
4. **Почему проверка до:** Умножение `res * 10` может переполнить 32-битное целое до сравнения с `INT_MAX`. Проверка `res > (INT_MAX - digit) // 10` эквивалентна `res * 10 + digit > INT_MAX`, но без риска переполнения при умножении.
5. **Особые случаи:** `s = "42"` → `42`. `s = "   -42"` → `-42` (пробелы + знак). `s = "4193 with words"` → `4193` (остановка на пробеле). `s = "words and 987"` → `0` (нет цифр после пробелов). `s = "-91283472332"` → `-2147483648` (clamping). `s = "+-12"` → `0` (после знака не цифра). `s = ""` → `0`. `s = "   "` → `0`.
6. **Время:** O(n), где n — длина строки, один проход.
7. **Память:** O(1) — только переменные `i`, `sign`, `res`, `digit`.

**Ключевой момент:** Отличие от задачи 7 — при переполнении возвращаем граничное значение (clamping), а не 0. Проверка знака происходит после пробелов, но до цифр.
