**Подход: Бинарный поиск**

1. **Идея:** Ищем наибольшее целое `ans`, такое что `ans² ≤ x`. Диапазон: `[0, x]`. На каждом шаге проверяем середину: если `mid² == x` → ответ. Если `mid² < x` → ищем справа. Если `mid² > x` → ищем слева.
2. **Логика:** `left = 0, right = x`. Пока `left ≤ right`: `mid = (left+right)//2`. Сравниваем `mid*mid` с `x`. Когда цикл заканчивается (`left > right`), `right` — последнее число с квадратом ≤ x.
3. **Почему бинарный поиск:** Перебор от 0 до x занял бы O(x). Бинарный поиск отбрасывает половину диапазона на каждом шаге — O(log x).
4. **Особые случаи:** `x = 0` или `1` → сразу `return x` (избегаем деления на ноль и лишних итераций).
5. **Время:** O(log x) — бинарный поиск по диапазону длиной x.
6. **Память:** O(1) — только переменные left, right, mid.

**Ключевой момент:** Когда `left > right`, `right` гарантированно указывает на наибольшее число, чей квадрат ≤ x. Потому что `right` — это последний `mid`, для которого `mid² < x`, после которого `left` обогнал `right`.
