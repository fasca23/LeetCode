**Подход: Отщипывание цифр с конца**

1. **Идея:** Берём последнюю цифру `x % 10`, добавляем к результату `rev = rev * 10 + digit`, убираем цифру `x //= 10`. Повторяем пока x ≠ 0.
2. **Логика:** Сохраняем знак, работаем с `abs(x)`. На каждом шаге: `digit = x % 10`, `rev = rev * 10 + digit`, `x //= 10`.
3. **Проверка переполнения:** ДО умножения `rev * 10` проверяем: если `rev > (INT_MAX - digit) // 10` — результат выйдет за границы, возвращаем 0.
4. **Почему проверка до:** Если сделать `rev * 10` без проверки, само умножение может переполниться (в языках с фиксированным int). В Python это не проблема, но для совместимости проверяем.
5. **Особые случаи:** `x = 0` → `0`. `x = 120` → `21` (ведущий ноль отбрасывается). `x = 1534236469` → переполнение → `0`.
6. **Время:** O(log₁₀ x) — число итераций равно количеству цифр в x.
7. **Память:** O(1) — только переменные rev, digit, sign.

**Ключевой момент:** Проверка `rev > (INT_MAX - digit) // 10` эквивалентна `rev * 10 + digit > INT_MAX`, но без риска переполнения при умножении.
