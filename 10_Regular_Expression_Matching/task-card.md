**Подход: Динамическое программирование**

1. **Идея:** Строим таблицу dp[i][j] — совпадают ли первые i символов s с первыми j символами p. Ответ — dp[m][n].
2. **Логика:** Три случая на каждом шаге:
   - Обычный символ/`.` — `dp[i][j] = dp[i-1][j-1]` если символы совпадают.
   - `*` — либо ноль повторений (`dp[i][j-2]`), либо одно+ (`dp[i-1][j]` если предыдущий символ совпадает).
3. **База:** Пустая строка и пустой шаблон — true. Пустая строка против `a*`, `a*b*` — true (через dp[0][j] = dp[0][j-2]).
4. **Почему DP:** Рекурсия без мемоизации — экспоненциальный взрыв из-за `*` (каждый раз два варианта). DP даёт O(m×n).
5. **Время:** O(m×n) — двойной цикл.
6. **Память:** O(m×n) — таблица. Можно оптимизировать до O(n) храня только две строки.

**Ключевой момент:** Для `*` проверяем два варианта: не использовать `x*` (dp[i][j-2]) ИЛИ использовать если символ совпал (dp[i-1][j]). Второй вариант берёт предыдущую строку той же колонки — это позволяет `*` сматчить много символов.
