**Подход: Динамическое программирование (двумерная DP-таблица)**

1. **Идея:** Строим таблицу `dp[i][j]`, где `i` — длина префикса строки `s`, `j` — длина префикса шаблона `p`. `dp[i][j] = True`, если `s[0:i]` совпадает с `p[0:j]`. Ответ — `dp[len(s)][len(p)]`.
2. **Логика:** Создаём таблицу `dp` размером `(m+1) x (n+1)`, где `m = len(s)`, `n = len(p)`. Заполняем всё `False`. База: `dp[0][0] = True` (пустая строка и пустой шаблон совпадают). Инициализация первой строки `dp[0][j]`: для `j` от 1 до n, если `p[j-1] == '*'`, то `dp[0][j] = dp[0][j-2]` (шаблон вида `a*` может соответствовать пустой строке — 0 вхождений). Основной цикл: `i` от 1 до m, `j` от 1 до n.
   - Если `p[j-1] == '*'`: рассматриваем два случая:
     - 0 вхождений символа перед `*`: `dp[i][j] = dp[i][j-2]`.
     - 1+ вхождений, если символ перед `*` совпадает с `s[i-1]` (или равен `'.'`): `dp[i][j] = dp[i][j] or dp[i-1][j]`.
   - Иначе (простой символ или `'.'`): если `p[j-1] == s[i-1]` или `p[j-1] == '.'`, то `dp[i][j] = dp[i-1][j-1]`.
3. **Проверка переполнения:** Не требуется, строковая задача.
4. **Почему DP, а не рекурсия:** Из-за оператора `*` (ноль или более вхождений) наивный рекурсивный перебор даёт экспоненциальное количество вариантов. DP сохраняет результаты подзадач и вычисляет ответ за полиномиальное время.
5. **Особые случаи:** `s = "aa"`, `p = "a"` → `False`. `s = "aa"`, `p = "a*"` → `True` (`*` делает 2 вхождения 'a'). `s = "ab"`, `p = ".*"` → `True` (`'.'` → 'a', `*` повторяет `'.'` для 'b'). `s = "aab"`, `p = "c*a*b"` → `True` (`c*` — 0 вхождений, `a*` — 2, `b` — 1). `s = ""`, `p = ".*"` → `True`. `s = "mississippi"`, `p = "mis*is*p*."` → `False`.
6. **Время:** O(m * n), где m = len(s), n = len(p) — двойной цикл по таблице.
7. **Память:** O(m * n) для хранения DP-таблицы (можно оптимизировать до O(n), храня только две строки).

**Ключевой момент:** Обработка `*`: либо пропускаем символ перед `*` (`dp[i][j-2]` — 0 вхождений), либо используем его повторно, если есть совпадение (`dp[i-1][j]` — 1+ вхождений). Инициализация первой строки для пустой `s` критична.
