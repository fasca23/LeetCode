**Подход: Два указателя с переключением**

1. **Идея:** Указатели pA и pB идут по A и B. Когда один доходит до конца — переходит на голову другого списка. К моменту встречи оба проходят одинаковое расстояние (lenA + lenB).
2. **Логика:** `pA, pB = headA, headB`. Пока `pA != pB`: `pA = pA.next if pA else headB`, `pB = pB.next if pB else headA`. Вернуть `pA` (либо узел пересечения, либо None).
3. **Время:** O(n + m) — каждый указатель проходит оба списка не более одного раза.
4. **Память:** O(1) — два указателя.

**Ключевой момент:** Переключение на другой список выравнивает длины. Указатели проходят одинаковый путь и встречаются ровно на пересечении.
