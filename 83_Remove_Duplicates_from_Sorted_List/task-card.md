**Подход: Один проход с пропуском**

1. **Идея:** Идём по списку. Если следующий узел равен текущему — меняем ссылку `current.next = current.next.next`, пропуская дубликат. Иначе переходим к `current.next`.
2. **Логика:** Указатель `current` идёт от головы. Пока `current` и `current.next` существуют: сравниваем их значения. Равны → пропускаем `next`. Не равны → двигаем `current` дальше.
3. **Почему список не ломается:** Мы меняем только ссылки `next`, не трогая значения. Пропущенные узлы остаются в памяти, но в список больше не входят.
4. **Особые случаи:** Пустой список или из одного элемента → сразу возвращаем `head`. Все элементы одинаковые → остаётся только первый.
5. **Время:** O(n) — один проход по всем узлам.
6. **Память:** O(1) — только указатель `current`.

**Ключевой момент:** После пропуска дубликата `current` НЕ двигается. Потому что следующий узел может тоже быть дубликатом (например, `[1,1,1]`). Двигаем `current` только когда значения разные.
